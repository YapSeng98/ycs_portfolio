ServiceNow × Claude AI — Natural Language Interface

A self-built web application that bridges ServiceNow with Claude AI (Anthropic API), letting users interact with ServiceNow records using plain English commands instead of navigating the portal. This project explores the potential of large language models as an intelligent interface layer over enterprise REST APIs — combining my ServiceNow expertise with hands-on AI agent development.

  System Architecture

Web App HTML / CSS / JS
⟶ user query
Proxy Layer Node.js / Express
⟶ prompt + context
Claude AI Anthropic API
⟶ API params
ServiceNow Scripted REST API
Response path: ServiceNow raw data → Claude AI formats → Web App displays
1

User Query — User types a plain English command in the web interface, e.g. "Show all open P1 incidents assigned to my team this week" or "Create a high priority incident for the email server being down."

2

Secure Proxy — The Node.js proxy layer receives the request and forwards it to Claude API with a structured system prompt containing ServiceNow API context, table schemas, and output format rules — keeping credentials off the browser.

3

Claude AI — Intent Parsing — Claude interprets the user's intent and outputs a structured JSON object containing the target ServiceNow table, encoded filter query, selected fields, and HTTP method (GET / POST / PATCH).

4

ServiceNow REST API Call — The proxy uses Claude's output to construct and execute the ServiceNow Scripted REST API call, retrieving or mutating the relevant records on the developer instance (PDI).

5

Claude AI — Response Formatting — The raw API JSON is passed back to Claude, which formats it into a clean, human-readable summary, table, or confirmation message tailored to what the user asked.

What It Does

  • Natural language interface to query, create, and update ServiceNow records without portal access
  • Claude AI dynamically constructs the correct ServiceNow table, filter query, and REST method
  • Supports incident queries, change request creation, and knowledge base search
  • AI-formatted responses — readable summaries and tables instead of raw JSON
  • Live integration with a personal ServiceNow developer instance (PDI)
  • Secure Node.js proxy layer keeps Anthropic and ServiceNow credentials off the client
  • Error handling with fallback messages when Claude output fails validation

What I Learned

  • How to craft effective system prompts that constrain Claude's output to valid API parameters
  • ServiceNow Scripted REST API — creating custom endpoints, encoded filters, and field queries
  • Designing a reliable JSON output schema for LLMs to follow consistently across varied inputs
  • Building a backend proxy to orchestrate multi-step AI + API calls securely
  • Handling edge cases where AI output must be validated before hitting external systems
  • The practical power — and limits — of combining LLMs with structured enterprise REST APIs
